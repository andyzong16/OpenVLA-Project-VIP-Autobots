Detailed Description

High-level interface to the MyCobot 280 robot arm. 

@class MC280
@brief High-level controller for the MyCobot 280 arm.
@details
Handles hardware initialization, motion commands, basic kinematics calculations, and PyBullet visualization.

@var self.port            Serial device path.
@var self.baud            Serial baud rate.
@var self.alpha_home      Home joint angles [deg] in URDF joint order.
@var self.alpha_sleep     Folded/"sleep" joint angles [deg].
@var self.alpha_limits    Per-joint min/max limits [deg], shape 2×6.
@var self.alpha_limits_rad Per-joint min/max limits [rad], shape 2×6.
@var self.joint_names     Joint name list matching URDF order.
@var self.max_speed       Max joint speed scale (device units, 1–100 or mapped).
@var self.mc              Low-level MyCobot280 API handle (connected, powered on).
@var self.package_root    Root path of the 'mc280' package.
@var self.model_path      URDF path used for PyBullet FK/IK.

◆ measure_flange()

def measure_flange (   self) 
@brief Query MyCobot for the flange pose as SE3.Homog.
@param mc  MyCobot280 instance.
@return Flange pose in the base frame as [x_mm, y_mm, z_mm, rx_deg, ry_deg, rz_deg].

The base frame is defined as:
- Display on the robot base faces the negative X direction.
- To the robot's left is the positive Y direction.
- Upward from the base is the positive Z direction

The base frame origin is at the XY center of the base. The Z=0 reference is the bottom edge of the base display panel.

@brief Move the gripper to a specified open percentage.

Sends a command to the gripper to reach the desired opening
percentage and blocks until the gripper finishes moving,
printing "moving" while it is in motion.

@param pct_open Desired opening of the gripper in percent:

    - 100 = fully open

The speed and force parameters passed to
mc.set_gripper_value() are fixed at 10 and 1 respectively.

def set_joints (   self,

@brief Move to a target joint configuration in a specified time. 
@param alpha  Target joint angles (deg).
@param time   Desired motion time (s). Must be > 0.
        Used to compute speed as:
        speed = (max_delta / time) / max_speed * 100,
        where max_delta is the largest joint change (deg).
        Clamped to [1,100].
@throws ValueError if time <= 0.
